## Top 10 Blockers

### 1. [CRITICAL] Global permit_all_parameters Disables Strong Parameters
**Domain:** Rails, Infrastructure
**Files:** `config/initializers/permit_all_parameters.rb`, `app/controllers/application_controller.rb`
**Impact:** Any authenticated user can overwrite any attribute on any model, including admin flags and subscription status.
**Fix:** Remove the initializer and add explicit `params.permit()` to every controller action.

### 2. [CRITICAL] SQL Injection in BoardDownstreamButtonSet.reconcile
**Domain:** Rails
**File:** `app/models/board_downstream_button_set.rb:690`
**Impact:** Arbitrary SQL execution, potential full database compromise.
**Fix:** Replace `where("id > #{start_id}")` with `where('id > ?', start_id)`.

### 3. [CRITICAL] No CI/CD Pipeline
**Domain:** Infrastructure
**Impact:** Code auto-deploys to production without tests, linting, or security scans. Regressions reach users immediately.
**Fix:** Implement GitHub Actions with rspec, ember test, bundle audit, npm audit gates.

### 4. [CRITICAL] Rack CVEs (CVE-2026-22860, CVE-2026-25500)
**Domain:** Dependencies
**Package:** rack 2.2.21
**Impact:** Directory traversal (CVSS 7.5) and stored XSS.
**Fix:** `bundle update rack` (to 2.2.22+).

### 5. [HIGH] Access Tokens Leaked in Logs
**Domain:** Privacy, Rails
**Files:** `app/models/api_call.rb:17`, `app/controllers/session_controller.rb:531`, `app/controllers/application_controller.rb:69`, `app/frontend/app/services/session.js`
**Impact:** Token exposure enables account takeover; violates GDPR data minimization.
**Fix:** Hash tokens before storage, remove console.log, set production log level to INFO.

### 6. [HIGH] PII Stored Indefinitely Without Retention Policy
**Domain:** Privacy
**Files:** `app/models/log_session.rb`, `app/models/device.rb`
**Impact:** IP addresses, user agents, and geolocation linked to communication records stored forever. GDPR violation.
**Fix:** Implement IP anonymization at collection and scheduled data purge jobs.

### 7. [HIGH] Database Open to All IPs (0.0.0.0/0)
**Domain:** Infrastructure
**Impact:** Both Postgres databases accessible from any IP worldwide.
**Fix:** Restrict to Render internal network IPs.

### 8. [HIGH] No Content Security Policy
**Domain:** Infrastructure, Rails
**Impact:** No defense against XSS, clickjacking, or script injection.
**Fix:** Create `config/initializers/content_security_policy.rb` with strict policy.

### 9. [HIGH] sendAction() Blocks Ember 4.0 Upgrade
**Domain:** Ember
**Count:** 75 occurrences in 21 files
**Impact:** sendAction was removed in Ember 4.0 -- must be replaced before any version upgrade.
**Fix:** Replace with closure actions.

### 10. [HIGH] lodash Prototype Pollution (CVE-2025-13465)
**Domain:** Dependencies
**Package:** lodash 4.17.21
**Impact:** Prototype pollution via _.unset and _.omit (CVSS 7.2).
**Fix:** Upgrade to lodash 4.17.23 in `app/frontend/package.json`.

## Cross-Cutting Themes

### 1. Token and Credential Exposure
Access tokens appear in API logs, console output, Rails debug logs, and controller logs. Combined with permit_all_parameters, leaked tokens grant unrestricted write access to all model attributes.

### 2. Missing Security Boundaries
No Strong Parameters, no CSP, no password complexity, no session timeout, database open to all IPs, no CI/CD security gates. The application lacks defense-in-depth.

### 3. Unmaintained and Vulnerable Dependencies
Rack has active CVEs, lodash has prototype pollution, Ember 3.28 is EOL, and multiple gems are abandoned.

### 4. PII Accumulation Without Governance
IPs, user agents, geolocation, emails, and tokens stored indefinitely with no retention policies, no scheduled purges, and incomplete consent mechanisms.

### 5. Technical Debt Blocking Modernization
The Ember frontend is entirely legacy-pattern code (484 classic files, "extreme" migration effort). Rails models skip ApplicationRecord. Schema is out of sync.

### 6. No Quality Gates in Deployment
Code deploys without tests, linting, or security scanning. Low frontend test coverage. No documented rollback or backup strategy.

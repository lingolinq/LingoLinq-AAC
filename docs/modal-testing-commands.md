### Notes

- Some modals require data to render (e.g., `share-board` needs a board, `button-settings` needs a button). They will show empty/error state without arguments — that's expected for visual style checking.
- `subscribe` and `terms-agree` are uncloseable — use `modal.close()` or refresh the page.
- `force-logout` has no close button (intentional — forces re-login).
- `big-button` and `timer` are full-screen interactive modals with no header (intentional).

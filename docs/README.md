# LingoLinq AAC Documentation 📚

Complete documentation for LingoLinq AAC - an AI-enhanced Augmentative and Alternative Communication platform.

## Documentation Structure

### 📁 ai/
AI assistant instructions and workflows:
- `CLAUDE.md` - Claude AI assistant setup and instructions
- `GEMINI.md` - Gemini AI assistant setup and instructions
- `workflows/` - AI development workflows and coordination

### 📁 development/
Core development documentation:
- `AI_DEVELOPMENT_GUIDE.md` - **Comprehensive AI assistant setup and workflows**
- `API_DOCUMENTATION.md` - Complete API reference
- `SETUP.md` - Development environment setup
- `CONTRIBUTING.md` - Contribution guidelines
- `CLEANUP.md` - Code cleanup and maintenance
- `CODE_INVESTIGATION.md` - Code analysis and investigation notes
- `TRANSLATIONS.md` - Internationalization and localization

### 📁 planning/
Project planning and strategic documentation:
- `roadmaps/` - Major project roadmaps and audit reports
- `features/` - Feature-specific planning (LLM inflections, privacy)
- `integrations/` - External integration planning (OpenAAC, standards)
- `MANAGEMENT_UPDATE_2025.md` - Management progress updates
- `PROGRESS_SUMMARY_2025.md` - Technical progress summaries
- `premium_features.md` - Premium feature planning
- `summary-all.md` - Comprehensive project overview

### 📁 architecture/
System architecture and design:
- `UX-UI-Modernization-Plan.md` - User experience modernization strategy

### 📄 Root Level Files
- `CHANGELOG.md` - Version history and release notes

## Getting Started

### For New Contributors
1. Start with `planning/summary-all.md` for project overview
2. Review `development/SETUP.md` for environment setup
3. Check `development/CONTRIBUTING.md` for contribution guidelines

### For AI Development  
1. **Start here**: `development/AI_DEVELOPMENT_GUIDE.md` - Comprehensive setup and workflows
2. See `ai/CLAUDE.md` or `ai/GEMINI.md` for assistant-specific setup
3. Review `ai/workflows/` for development coordination
4. Check `planning/features/` for AI feature planning

### For Security and Compliance
1. Review `planning/roadmaps/01-Security-Audit-Report.md`
2. Check `planning/roadmaps/02-Compliance-Requirements.md`
3. See `planning/features/INFLECTION_PRIVACY_ARCHITECTURE.md`

### For Technical Architecture
1. Read `architecture/UX-UI-Modernization-Plan.md`
2. Check `planning/roadmaps/03-MVP-Modernization-Plan.md`
3. Review `development/API_DOCUMENTATION.md`

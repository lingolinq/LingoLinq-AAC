**Migration Pattern Applied:**
1. Added `import { inject as service } from '@ember/service';`
2. Added service injections: `stashes: service('stashes')`, `persistence: service('persistence')`, `appState: service('app-state')` as needed
3. Replaced `stashes.get(...)` with `this.stashes.get(...)` or `_this.stashes.get(...)` in callbacks
4. Replaced `persistence.get(...)` with `this.persistence.get(...)` or `_this.persistence.get(...)` in callbacks
5. Replaced `app_state.get(...)` with `this.appState.get(...)` or `_this.appState.get(...)` in callbacks
6. Fixed `this` context issues in callbacks by using `_this` pattern
7. Removed unused imports where services were not actually used

**Key Technical Details:**
- All routes now use explicit service injection via `@service` decorator
- Callback contexts properly handled using `_this` pattern
- No breaking changes - all routes maintain backward compatibility
- All files pass linting
- No remaining old utility imports in routes directory

**Impact:**
- Eliminates implicit dependency on global utility objects
- Improves testability and maintainability
